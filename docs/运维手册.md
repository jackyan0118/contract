# 价格附件生成系统 - 运维手册

## 系统概述

### 系统简介

价格附件生成系统是一个基于 FastAPI 框架开发的 RESTful API 服务，用于自动生成 Word 格式的价格附件文档。

### 技术栈

| 组件 | 技术 | 版本要求 |
|------|------|----------|
| Web 框架 | FastAPI | 0.100+ |
| Python | Python | 3.10+ |
| 数据库 | Oracle (python-oracledb) | Thin 模式 |
| 文档生成 | python-docx | 0.8+ |
| 日志 | Python logging | 内置 |

### 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端    │────▶│  FastAPI    │────▶│  Oracle     │
│  (调用API)  │     │  应用服务   │     │  数据库     │
└─────────────┘     └─────────────┘     └─────────────┘
                          │
                          ▼
                   ┌─────────────┐
                   │  Word 模板  │
                   │  文档生成   │
                   └─────────────┘
```

### 核心功能

1. **报价单查询**: 根据报价单号(WYBS)查询报价数据
2. **模板匹配**: 根据产品分类自动匹配生成模板
3. **文档生成**: 生成 Word 格式价格附件
4. **批量处理**: 支持批量生成多个报价单
5. **任务管理**: 异步任务状态追踪

## 快速开始

### 环境要求

- Python 3.10+
- Oracle Database 11g+
- 4GB+ RAM
- 10GB+ 磁盘空间

### 安装部署

#### 1. 克隆项目

```bash
git clone <repository-url>
cd contract
```

#### 2. 创建虚拟环境

```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或
venv\Scripts\activate     # Windows
```

#### 3. 安装依赖

```bash
pip install -e ".[dev]"
```

#### 4. 配置环境变量

```bash
# 复制配置模板
cp config/settings.yaml.example config/settings.yaml

# 编辑配置文件
vim config/settings.yaml
```

#### 5. 启动服务

```bash
# 开发环境
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000

# 生产环境（推荐使用 gunicorn）
gunicorn src.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

## 配置说明

### 配置文件结构

```yaml
# config/settings.yaml
app:
  name: 价格附件生成系统
  debug: false                    # 生产环境必须为 false
  version: 1.0.0
  host: 0.0.0.0
  port: 8000

database:
  host: oracle.example.com
  port: 1521
  service_name: orcl
  user: price_app
  password: ${DB_PASSWORD}       # 使用环境变量
  min_pool_size: 2
  max_pool_size: 10

logging:
  level: INFO                     # 生产环境建议 INFO
  file_path: logs/app.log
  max_bytes: 10485760            # 10MB
  backup_count: 5

api:
  api_key: ${API_KEY}             # 必须设置
  rate_limit: 100                 # 每分钟请求限制
```

### 环境变量

| 变量名 | 必填 | 说明 | 默认值 |
|--------|------|------|--------|
| `DB_PASSWORD` | 是 | Oracle 数据库密码 | - |
| `API_KEY` | 是 | API 认证密钥 | - |
| `LOG_LEVEL` | 否 | 日志级别 | INFO |
| `APP_DEBUG` | 否 | 调试模式 | false |

## API 接口

### 基础信息

| 项目 | 说明 |
|------|------|
| 基础路径 | `/api/v1` |
| 文档地址 | `/docs` |
| 健康检查 | `/api/v1/health` |

### 接口列表

#### 1. 健康检查

```bash
GET /api/v1/health
```

响应示例：
```json
{
  "success": true,
  "data": {
    "status": "healthy",
    "database": "connected",
    "timestamp": "2026-03-17T10:00:00"
  }
}
```

#### 2. 生成价格附件

```bash
POST /api/v1/generate
X-API-Key: your-api-key

{
  "wybs": "WYBS20260310001"
}
```

#### 3. 批量生成

```bash
POST /api/v1/batch
X-API-Key: your-api-key

{
  "wybs_list": ["WYBS001", "WYBS002"],
  "is_async": true
}
```

#### 4. 查询模板列表

```bash
GET /api/v1/templates
X-API-Key: your-api-key
```

#### 5. 任务状态查询

```bash
GET /api/v1/tasks/{task_id}
X-API-Key: your-api-key
```

## 运维操作

### 服务管理

#### 启动服务

```bash
# 使用 systemd (推荐)
sudo systemctl start price-app

# 或使用 supervisor
supervisorctl start price-app
```

#### 停止服务

```bash
sudo systemctl stop price-app
```

#### 重启服务

```bash
sudo systemctl restart price-app
```

#### 查看服务状态

```bash
sudo systemctl status price-app
```

### 日志管理

#### 查看实时日志

```bash
# 应用日志
tail -f logs/app.log

# 系统日志
journalctl -u price-app -f
```

#### 日志轮转

系统默认配置：
- 单文件大小：10MB
- 保留文件数：5 个
- 总容量：50MB

如需调整，修改 `config/settings.yaml`：

```yaml
logging:
  max_bytes: 20971520    # 20MB
  backup_count: 10       # 保留10个
```

### 数据库连接池

#### 监控连接状态

```bash
# 查看当前连接数
SELECT COUNT(*) FROM v$session WHERE username = 'PRICE_APP';

# 查看连接池配置
SELECT pool_min, pool_max FROM dba_resource_manager_plan;
```

#### 调整连接池大小

在 `config/settings.yaml` 中修改：

```yaml
database:
  min_pool_size: 5      # 最小连接数
  max_pool_size: 20     # 最大连接数
```

### 模板管理

#### 模板存放位置

```
docs/template/
├── 模板1.docx
├── 模板2.docx
├── 模板3.1.docx
└── ...
```

#### 模板规则配置

```
config/template_rules.yaml
```

配置说明：
- `产品细分`: 产品分类（通用生化试剂、酶免试剂等）
- `是否集采`: 是否集中采购（0=是，1=否）
- `排序规则`: 排序方式

## 故障处理

### 常见问题

#### 1. 服务启动失败

**症状**: `Connection refused` 或端口被占用

**排查步骤**:
```bash
# 检查端口占用
lsof -i :8000

# 检查配置文件
python -c "from src.config.settings import settings; print(settings)"
```

#### 2. 数据库连接失败

**症状**: `ORA-` 错误码

**排查步骤**:
```bash
# 测试数据库连接
python -c "from src.database.connection import get_connection_pool; p = get_connection_pool(); p.initialize(); print('OK')"

# 检查网络连通性
tnsping oracle.example.com
```

#### 3. 模板匹配失败

**症状**: 返回空模板列表

**排查步骤**:
```bash
# 检查模板文件是否存在
ls -la docs/template/

# 检查规则配置
cat config/template_rules.yaml | python -m json.tool
```

#### 4. 文档生成失败

**症状**: 生成接口返回错误

**排查步骤**:
```bash
# 检查日志
tail -n 100 logs/app.log | grep ERROR

# 检查临时目录权限
ls -la /tmp
```

### 紧急恢复

#### 服务无法启动

```bash
# 1. 查看详细错误
journalctl -xe --no-pager

# 2. 检查依赖
pip list | grep -E "(fastapi|oracledb|docx)"

# 3. 重置数据库连接
sudo systemctl restart oracle
```

#### 数据库连接池耗尽

```bash
# 1. 增加连接池大小
# 2. 重启服务释放连接
sudo systemctl restart price-app
```

## 性能优化

### 接口性能

| 接口 | 预期响应时间 | 优化建议 |
|------|-------------|----------|
| /health | < 100ms | 缓存数据库状态 |
| /generate | < 5s | 优化模板匹配逻辑 |
| /batch | < 30s | 使用异步处理 |

### 数据库优化

```sql
-- 创建索引
CREATE INDEX idx_wybs ON uf_htjgk(wybs);
CREATE INDEX idx_cpxf ON uf_htjgk_dt1(cpxf);
```

建议对以下内容添加缓存：
- 模板规则加载
- 产品分类映射
- 字段配置
