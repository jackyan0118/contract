# 价格附件生成系统 - 工作流程文档

本文档描述从 API 接收到 Word 文档生成的完整工作流程。

## 整体架构

系统采用典型的 **ETL** 架构：
- **E**xtract (查询): 从 Oracle 数据库获取报价单数据
- **T**ransform (转换): 数据类型转换和字段映射
- **L**oad (加载): 填充到 Word 模板生成文档

## 核心模块说明

### 1. API 入口 (generate.py)

**文件**: `src/api/routes/generate.py`

主要流程:
1. 验证 API Key
2. 接收 `wybs` (报价单号)
3. 查询主表和明细表数据
4. 数据转换 (DataTransformer)
5. 提取产品细分、定价组、是否集采等信息
6. 使用 `RuleLoader` 加载规则并匹配模板
7. 调用 `DocumentGenerator` 生成文档
8. 打包文件 (FilePacker)
9. 返回 Base64 编码的 Word/ZIP 文件

### 2. 数据查询 (queries/)

| 文件 | 表 | 用途 |
|------|-----|------|
| `quotation.py` | `uf_htjgkst` | 报价单主表 |
| `quotation_detail.py` | `uf_htjgkst_dt1` | 报价单明细表 |

**明细表关联查询**:
- `UF_CPKXMJC` - 获取项目简称 (XMJC)
- `UF_CPXF` - 获取产品细分名称和 BM 编码

### 3. 数据转换 (transformers/)

**文件**: `src/transformers/data_transformer.py`

- 使用 `FieldMappingConfig` 从 `config/field_mapping.yaml` 加载字段映射
- 将 Oracle 字段名转换为内部字段名
- 类型转换:
  - 价格字段 → `Decimal` (2位小数)
  - 整数字段 → `int`

### 4. 模板匹配 (matchers/ + services/)

**文件**:
- `src/matchers/template_matcher.py`
- `src/services/rule_loader.py`

**匹配逻辑**:
1. 从 `config/template_rules.yaml` 加载模板规则
2. 从明细数据提取产品细分信息（CPXF_BM, CPXF_NAME 等，已通过 SQL 关联查询获取）
3. 直接使用 `CPXF_BM` 匹配模板规则中的 `产品细分编号`

### 5. 文档生成 (generators/)

**文件**: `src/generators/document_generator.py`

详细步骤:
1. 加载模板元数据配置 (YAML)
2. 复制模板到输出目录
3. 填充段落占位符
4. 填充表格 (关键步骤)
5. 处理话术
6. 修复表格布局
7. 保存文档

### 6. 数据填充 (fillers/)

| 模块 | 职责 |
|------|------|
| `data_filler.py` | 数据过滤、条件匹配、变量替换 |
| `row_expander.py` | 表格行扩展、单元格合并 |
| `speech_processor.py` | 话术处理、互斥组逻辑 |
| `constants.py` | 常量定义 |

**RowExpander 关键功能**:
- 动态插入数据行
- 保留话术行在表格末尾
- 按折扣率垂直合并单元格 (vMerge)

**SpeechProcessor 关键功能**:
- 条件话术: 根据数据条件决定是否显示
- 固定话术: 直接添加
- 互斥组: 同组话术只显示一个

### 7. 异常处理 (exceptions/)

**文件**:
- `src/exceptions/base.py` - 基础异常类
- `src/exceptions/document.py` - 文档生成异常

### 8. 配置加载

**TemplateLoader** (`src/config/template_loader.py`):
- 从 `config/template_metadata/templates/{id}.yaml` 加载模板元数据
- 包括: 表格列配置，明细过滤规则、话术配置、产品匹配规则

**RuleLoader** (`src/services/rule_loader.py`):
- 从 `config/template_rules.yaml` 加载模板规则
- 支持热更新 (watchdog)

## 配置文件说明

### config/template_rules.yaml

模板匹配规则配置文件，包含:
- **产品细分映射**: cpxf_name → bm 编码
- **模板规则**: 模板匹配条件

### config/template_metadata/templates/{模板ID}.yaml

单个模板的元数据配置，包含:
- **detail_filter**: 明细数据过滤条件
- **table.columns**: 表格列配置
- **paragraph_placeholders**: 段落占位符映射
- **speeches**: 话术配置
- **product_matching**: 产品匹配规则（用于模板6.1等）

## 关键配置项说明

### detail_filter (明细过滤)

用于从报价单明细中筛选符合条件的数据行。

```yaml
detail_filter:
  condition_groups:
  - conditions:
    - field: CPXF
      operator: '='
      value: '150'
    - field: DJZMC
      operator: contains
      value: 肾功和心肌酶
    name: 卓越生化肾功心肌酶
```

**字段说明**:
- `field`: 字段名（支持 CPXF, CPXF_BM, DJZMC 等）
- `operator`: 操作符 (=, !=, contains, >, < 等)
- `value`: 比较值

### product_matching (产品匹配)

用于标准化产品名称（模板6.1）。

```yaml
product_matching:
  products:
    - name: β2-微球蛋白测定试剂盒(胶乳增强免疫比浊法)
      seq: 1
    - name: 肌酐检测试剂盒（肌氨酸氧化酶法）
      seq: 2
```

### speeches (话术配置)

用于在表格下方或段落中显示话术。

```yaml
speeches:
- id: 话术1
  type: fixed
  position: after_table
  content: "本报价含税..."
- id: 话术2
  type: conditional
  conditions:
  - field: DJZMC
    operator: contains
    value: 特殊
  content: "特殊定价组话术..."
  mutex_group: group1
```

## API 接口说明

### POST /api/generate

单文件生成接口。

**请求参数**:
```json
{
  "wybs": "WYBS20250316001"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "filename": "价格附件.docx",
    "file_base64": "UEsDBBQABgA...",
    "templates_used": ["模板6.1"]
  },
  "message": "生成成功"
}
```

### POST /api/generate/multi

多文件生成接口（返回 ZIP）。

### POST /api/batch/generate

批量生成接口。

### GET /api/templates

获取可用模板列表。
