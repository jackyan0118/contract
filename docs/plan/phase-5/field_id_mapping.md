## 配置示例

### 使用名称匹配（可读性好）
```yaml
- field: "DJZMC"
  operator: "contains"
  value: "肝功"
- field: "BNGHJLX"
  operator: "="
  value: "集采中标价"
```

### 使用ID匹配（更稳定）
```yaml
- field: "DJZ"
  operator: "="
  value: 66  # 通用生化-肝功
- field: "BNGHJLXZ"
  operator: "="
  value: 5  # 集采中标价
```

## 匹配优先级

代码匹配逻辑：
1. **数值类型value** → 优先匹配ID字段（如 DJZ, BNGHJLXZ）
2. **字符串类型value** → 匹配名称字段（如 DJZMC）
3. 如果ID字段匹配成功，使用ID查询结果；否则fallback到名称字段

这样既保证稳定性，又保持可读性。
